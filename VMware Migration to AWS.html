VMware Migration to AWS

← Back to Portfolio

Overview

Client: Logistics Firm

Proposal: Migrate the entire VMware vSphere platform to AWS due to VMware’s product lineup changes and rising licensing costs. AWS provides a more economical and scalable virtualization solution.

Key Objectives

  • Establish AWS Direct Connect and Site-to-Site VPN for secure connectivity.
  • Migrate VMs from VMware vSphere to AWS using VM Import/Export.
  • Replace legacy VM-based services with AWS native managed services (File Storage, Domain Control, DNS, etc.).

Existing Arrangement

  • 13 virtual machines hosted on VMware vSphere.
  • Functions include: Application servers (Accounting & HR), Domain Controller, DNS, SQL, and File servers.

Approach

1. Retire

  • Consolidate workloads and decommission redundant VMs to optimize resource usage.

2. Rehost (Lift & Shift)

  • Migrate VMs directly to AWS without modifying applications.

Custom AMI creation via VM Import/Export

  1. Export VMware VM as OVA or VMDK.
  2. Upload to Amazon S3.
  3. Convert using AWS VM Import/Export into an AMI.
  4. Launch EC2 instances from the AMI.

This method is particularly valuable for legacy operating systems (e.g., Windows Server 2003/2008) where official AMIs are unavailable.

3. Re‑platform (Modernize)

  • AWS Directory Service → Domain Controller
  • Amazon Route 53 → DNS
  • Amazon FSx → File server

File server data migration options

Option A: AWS Snowball Edge
  • Use case: Large-scale data migration.
  • Pros: No bandwidth impact, secure, scalable.
  • Cons: Physical shipping introduces delays.
Option B: AWS DataSync
  • Use case: Automated transfer with sufficient bandwidth.
  • Pros: Fast, supports incremental sync, integrates with S3/EFS/FSx.
  • Cons: Transfer costs; requires agent setup.

If a 1 Gbps circuit is unavailable, AWS Snowball Edge is recommended.

Key Benefits

  • Reduced licensing costs compared to VMware.
  • Improved scalability and flexibility with AWS managed services.
  • Optimized infrastructure by retiring redundant VMs.
  • Secure and efficient data migration options tailored to customer bandwidth constraints.

Features

  • Migration of 13 VMware vSphere VMs to AWS.
  • Use of AWS Direct Connect and Site-to-Site VPN for secure connectivity.
  • VM Import/Export for legacy OS workloads (e.g., Windows Server 2003/2008).
  • Adoption of AWS managed services:
    • AWS Directory Service (Domain Controller)
    • Amazon Route 53 (DNS)
    • Amazon FSx (File server)
  • Flexible data migration options: AWS Snowball Edge (large datasets) or AWS DataSync (bandwidth‑based automation).
  • Optimization through VM retirement and consolidation to reduce overhead.

Lessons

  • VM Import/Export proved especially valuable for legacy systems where AWS AMIs were unavailable.
  • Early consideration of bandwidth limitations was critical in choosing between Snowball Edge and DataSync.
  • Consolidation of workloads reduced complexity and improved efficiency.
  • Leveraging AWS managed services minimized operational overhead compared to maintaining multiple EC2 instances.
  • A clear migration strategy (Retire → Rehost → Re‑platform) ensured smooth transition and avoided unnecessary relocation efforts.
