Overview

This project demonstrates how AWS serverless services can be leveraged to deliver scalable, cost‑efficient APIs. It highlights my role as a presales engineer in designing and presenting solutions from a system integrator perspective.

Existing Arrangement

The client initially relied on monolithic applications hosted on traditional infrastructure, facing challenges in scalability, cost, and operational overhead.

Approach

I conducted requirement analysis, mapped workloads to AWS services, and proposed a phased migration strategy. The focus was on reducing operational burden while ensuring compliance and resilience.

Final Architecture

The solution integrates AWS Lambda for compute, API Gateway for endpoint management, and DynamoDB for persistence. CloudWatch and IAM policies ensure observability and security.

Features

  • Fully serverless, pay‑per‑request model
  • Integrated with DynamoDB for persistence
  • API Gateway for RESTful endpoints
  • CloudWatch monitoring and IAM‑based security
  • Scalable and cost‑optimized design

Hindsight

Key lessons learned include the importance of early stakeholder alignment, clear documentation of service limits, and proactive monitoring setup. Future iterations could explore AWS Step Functions for orchestration and enhanced CI/CD pipelines.
