Synology Cloud Backup for Data Resilience

← Back to Portfolio

Overview

Client: Office NAS environment with limited IT budget

Proposal: Implement a resilient, cost-effective backup strategy using onsite external HDD for primary backups and AWS S3 for offsite secondary backups via Synology Hyper Backup.

Key Objectives

  • Ensure offsite data protection to mitigate local outages or device failure.
  • Maintain cost efficiency by choosing appropriate Amazon S3 storage classes.
  • Enable encrypted backups with client-side encryption.

Existing Arrangement

  • NAS Model: Synology DS918+ with SHR configuration.
  • Capacity: 4 TB usable storage.
  • Local Protection: RAID configured; periodic USB external HDD backups.

Approach

Backup strategy

  • Primary: Onsite external HDD for fast local recovery.
  • Secondary: Offsite AWS S3 via Synology Hyper Backup for disaster resilience.

AWS side

  • S3 bucket: Create a dedicated bucket for NAS backups.
  • IAM user: Provision Access Key and Secret Access Key with least-privilege policy.

Synology NAS side

  • Hyper Backup: Configure backup tasks targeting AWS S3.
  • Encryption: Enable client-side encryption for secure data at rest and in transit.
  • Execution: Schedule and run backup jobs with verification.

Features

  • 3‑2‑1 backup rule: 3 copies, 2 different media, 1 offsite.
  • Contingency plan: Defined recovery steps for NAS failure scenarios.
  • Encryption: Protects data during offsite transfer and storage.
  • Cost control: S3 class selection aligned with access patterns.

Technical Considerations

Primary backup job

  • Window: Schedule during low-usage hours for minimal impact.
  • Method: Incremental backups to external HDD.

Secondary backup job

  • Window: Nightly or off‑peak to optimize bandwidth.
  • Method: Incremental, versioned backups to S3 via Hyper Backup.

Lessons

Insight: The selected Amazon S3 storage class materially affects ongoing costs and resilience. Intelligent‑Tiering helps when access is unpredictable, while One Zone‑IA minimizes cost for easily re‑creatable secondary copies.

Amazon S3 storage classes comparison

Storage Class Use Case Features Cost (USD/month)
S3 Standard Frequently accessed data (websites, apps, gaming) High durability, availability, performance 25.60
S3 Intelligent‑Tiering Unknown or unpredictable access patterns Auto‑moves data to cost‑effective tiers 25.76
S3 Standard‑IA Long‑term storage, backups, DR Lower cost for infrequent access; rapid retrieval 14.13
S3 One Zone‑IA Secondary copies; easily re‑creatable data Stored in a single Availability Zone 11.26
