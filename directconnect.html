Direct Connect and On-Prem VM Migration to AWS

← Back to Portfolio Overview Approach Features Lessons

Overview

Client: Financial Security Firm

Proposal: Migration of internal systems from on-premises data center to AWS.

  • Establish AWS Direct Connect and Site-to-Site VPN
  • Migration of VMs from Stratus everRun Virtualization Platform to AWS
  • Deployment of resilient architecture on AWS

Existing Arrangement

The customer’s financial application runs on Stratus everRun Virtualization Platform.

  • Hardware nearing end-of-life
  • CentOS-based OS, shifting to Ubuntu due to CentOS EOL
  • All hardware are placed at Data Center

Approach

Goal: Recreate comparable resilience on AWS using native services.

Step 1: VPC and Subnet Setup

  • Create separate VPCs for Production and UAT
  • Configure subnets across multiple AZs

Step 2: Routing Setup

Site-to-Site VPN: VGW, Customer Gateway, IPsec tunnel, routing, connectivity test.

Direct Connect: Private VIF, BGP peering, failover and throughput testing.

Note: Provisioning requires months of lead time.

Step 3: Instance Setup

  • Deploy EC2 (Windows Server) in Production and UAT VPCs
  • Active-Passive failover across AZs
  • Amazon RDS for Oracle (Multi-AZ, License Included)

Step 4: Route 53 Setup

Configure DNS-level health checks for failover.

Step 5: Backup & Monitoring

  • AWS Backup snapshots for EC2 and RDS
  • CloudWatch monitoring and SNS notifications

Features

Reliability & Availability

  • Route 53 DNS failover
  • Multi-AZ EC2 and RDS
  • Automated backups

Security Enhancements

  • Encrypted connectivity via Direct Connect and VPN
  • Separate VPCs for Production and UAT

Cost Optimization

  • Reduced overhead with AWS managed services
  • Right-sized architecture

Business Value

  • Lower hardware footprint and colocation costs
  • Improved scalability

Cost & Security Considerations

  • Hardware replacement vs AWS migration
  • Security compliance evaluation

Estimate Summary: To provide a clearer view of pricing using AWS Pricing Calculator.

Customised Breakdown: AWS service-by-service Matrix with details

Hindsight & Lessons Learned

  • Differences between on-prem and AWS architectures
  • Connectivity from colocation to AWS was critical
  • ISP coordination essential for Direct Connect

Extra Note on Terminology

Fault-tolerant: Seamless continuation despite hardware failure, requiring specialized infra.

Resilient: Recovery and continuity via redundancy, automation, managed services. Balances availability, cost, scalability.

This migration emphasizes resilience over strict fault tolerance, aligning with cloud-native design principles.
